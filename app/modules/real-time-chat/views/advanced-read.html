$loaded()

  • Returns a promise which is resolved when the initial object data has been downloaded from the database. The promise resolves to the $firebaseObject itself.
  • As a shortcut, the resolve() / reject() methods can optionally be passed directly into $loaded():

  var db = firebase.database().ref();
  // firebase API
  db.on('value').then(function(data) {
    console.log(data);
  });
  // AngularFire $loaded
  var obj = $firebaseObject(db);
  obj.$loaded().then(function(data) {
    console.log(data === obj);
  });

Sometimes when reading data you need to react to the fact that the data has been retrieved. Both the Firebase API and AngularFire provide a mechanism to do this.

For Firebase, this is the default method -- either use the “once” method, or use “on” to read data and listen for updates, and then react to the data being loaded through the promise.

For AngularFire the promise is not necessary as the $firebaseObject or $firebaseArray resolve to the data itself automatically. However one of the recommended implementations for services methods when using AngularFire is to return a $loaded() method from an array or object so that consumers of that service may depend upon the promise as necessary to suit their needs.
