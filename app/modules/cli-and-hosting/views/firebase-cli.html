The database.rules.json file should contain an exact copy of the database Rules you have configured within the Firebase Console. Having these rules as part of your project is great because it allows you to version rules changes, and to include rules with the code updates that may necessitate the changes within your source control. However if you develop a workflow of entering rules using the Simulator in the Console it is it is also easy to end up deploying an update the blows away your rules causing hours of frustration.
