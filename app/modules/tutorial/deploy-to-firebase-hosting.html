Deploy to Firebase Hosting

Success Criteria

  • Installed Firebase CLI tools
  • Connect to Fireside through public Firebase URL

Instructions

  1. Install the firebase CLI tools
  2. Initialize the project to create your firebase.json file: $ firebase init
  3. Login to Firebase using the CLI: $ firebase login
  4. Copy your database Rules from the Firebase database console to your database.rules.json file
  5. Run the project build using Firebase: $ gulp build && firebase serve
  6. Test the project http://localhost:8080/
  7. Deploy the project: $ firebase deploy
  8. Open up the Hosting tab on your Firebase console to see the deployment history
  9. Click the Domain shortcut to launch the app on your public website
  10. aste the URL to your app in the Slack channel

Hints

  • Learn more about the Firebase CLI - https://firebase.google.com/docs/cli/
  • Install: $ npm install -g firebase-tools
  • Deploying to Firebase using the CLI will replace the database rules loaded in firebase with the ones you have saved in your local database.rules.json file, so be sure this file is up-to-date.

QA

Run the app on Firebase using your project prefix in the form of https://.firebaseapp.com/

For example, our project is available at https://fireside-73c10.firebaseapp.com/
